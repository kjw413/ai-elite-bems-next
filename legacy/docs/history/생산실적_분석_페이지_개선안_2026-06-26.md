## 1. 현재 페이지 진단

8개 섹션 + 데이터 흐름 확인 결과.

| 섹션 | 현재 | 문제 |
|---|---|---|
| 요약 KPI | 누계 계획/실적/진척률/품목수 | **진척률 착시** — `planned_qty`는 *월말까지 누계 계획*인데 월 중간에 보면 "경과일 실적 ÷ 한 달 계획"이라 구조적으로 낮게(예: 10일차 ~65%) 찍힘. 페이스 보정 없음 |
| 제품유형 도넛 | 실적/계획 비중 | 무난하나 정보량 낮음 |
| 일별 추이 | 제품유형별 라인 | OK |
| Top10 품목 | **실적 절대값 순** | 늘 같은 효자품목만 노출. 관리가 필요한 "미달/초과"가 안 보임 |
| 에너지믹스↔실적 보정 | 완제품/재공품/외주 분해 | 이 페이지의 강점, 유지 |
| 에너지 vs 생산량 | 원단위 | OK |
| 월별 누계 실적 추이 | `monthly_actual`(월별 **단일** 합계) | **제목·내용 불일치** — 누적이 아님 (`production_performance.py` 월별 추이 섹션) |
| What-if 시뮬레이터 | kg→전력/연료/용수 회귀 추정 | **생산실적 분석이 아니라 에너지 기획 도구** — 별도 룩업 의존, 입출력이 생산 맥락과 불일치 → 제거 결정 |

### 데이터 제약 (개선안의 전제)
`production_daily` 컬럼: `date · item_code · item_name · factory · category1(보관) · category2(제품) · planned_qty(월누계계획) · actual_qty(일실적)`.
→ **라인·설비·교대(shift)·정지사유·불량/수율·정격 capacity 데이터 없음.** 진짜 OEE는 현재 데이터로 산출 불가. 별도 `energy_daily`(전력/연료/용수/믹스kg)와 계획·재공품(WIP) 데이터는 활용 가능.

## 2. 실제 식품·제조 MES/EMI 생산실적 분석 화면 표준 기능

1. **OEE(설비종합효율)** = 가동률 × 성능 × 품질 + 6대 로스 분해
2. **계획 대비 실적** — 편차/달성률, 라인·교대·품목별 비교, **목표 대비 진도(pace)**
3. **수율/불량** — First Pass Yield, 불량률(라인·교대별)
4. **다운타임 분석** — 정지사유 분류 + **Top 5 로스 Pareto**
5. **품목/라인/교대 교차 비교**
6. **생산 리드타임·사이클타임/택타임**
7. **식품 특화** — Changeover(품목전환)·CIP(세척) 시간을 비계획 정지와 분리 집계

## 3. 우리 데이터로 가능한 것 / 추가 수집 필요

| 표준 기능 | 가능 여부 | 비고 |
|---|---|---|
| 계획 대비 실적 + **페이스 보정 진척률** | ✅ | 경과 영업일/총 영업일로 기대 진도 |
| **Burn-up S-커브** (일별 누적 실적 vs 누적 계획) | ✅ | "누계 추이"의 올바른 형태 |
| 달성률 랭킹 / 미달·초과 Top | ✅ | Top10 편차순 재정렬 |
| 생산 안정성(가동률 proxy) — 일 변동성(CV)·무생산일·요일패턴 | ✅ | 진짜 가동률 대체 지표 |
| 계획 정확도 = \|실적−계획\|/계획 | ✅ | 계획 수립 품질 |
| MoM / **YoY(전년 동월)** · 계절성 | ✅ | 빙과 계절성 큰 업종 핵심 |
| 에너지 원단위 ↔ 생산 mix 연계 | ✅ (일부 보유) | 강화 |
| 진짜 OEE (가동률·성능·품질) | ⚠️ 추가 수집 | 라인 가동시간·정격 capacity·불량수량 필요 |
| 다운타임 Pareto / FPY·불량률 | ⚠️ 추가 수집 | 정지사유·불량 데이터 |
| Changeover/CIP 시간 | ⚠️ 추가 수집 | 세척·전환 로그 |

## 4. 재설계안 (우선순위별)

### P0 — 즉시 (추가 데이터 불필요, 오해 해소 중심)

**① 월별 "누계" 차트 분리 (확정)**
- **(A) 이달 진도 — Burn-up S-커브 (신규):** X=일, 누적 실적선 vs 누적 계획 페이스선(`planned_qty`를 영업일 분배). 오늘 시점에서 페이스 대비 위/아래 즉시 판별. → 사용자가 기대한 "누계"의 실제 모습.
- **(B) 월별 실적 추이 (개명):** 기존 차트를 정직하게 "월별 실적 추이"로 변경 + **전년 동월(YoY) 오버레이**.

**② 페이스 보정 진척률 KPI**
- 진척률 옆에 **"기대 진도 대비"**(= 실적 ÷ (계획 × 경과영업일비율)) 표기, 100% 기준 앞섬/뒤짐 표시.
- **월말 착지 예상**(현재 페이스 외삽) 추가.

**③ Top10 품목 → "계획 미달 Top / 초과 Top" 탭**
- 실적 절대값 대신 편차(실적−계획) 또는 달성률 순. 관리 필요 품목이 상단으로.

**④ What-if 시뮬레이터 제거 (확정)**
- 생산 페이지에서 삭제. (kg→에너지 추정 활용처는 추후 에너지 페이지 등에서 재검토)

### P1 — 가치 높음 (데이터 있음)
- **⑤ 달성률 히트맵:** 공장 × 제품유형 매트릭스, 색=달성률 → 부진 조합 즉시 식별
- **⑥ 생산 안정성 패널(가동률 proxy):** 품목별 일 생산 CV, **무생산일 캘린더 히트맵**, 요일별 평균 → 결품/안정성 가시화
- **⑦ 계획 정확도 추적:** 계획-실적 오차 월별 추이 → 계획 품질 개선 루프

### P2 — 로드맵 (신규 데이터 수집 필요)
- **⑧ 경량 OEE:** 라인 가동시간·정격속도·불량수량 3개 컬럼만 수집하면 가동률·성능·품질 산출 가능 (전면 MES 없이 핵심 3지표만)
- **⑨ 다운타임 Pareto / 불량 Top:** 정지·불량 사유 코드 테이블 신설 시
- 식품 특화 **CIP/Changeover 시간**은 ⑧ 로스 분해의 하위로 편입

## 5. 권고

- 임팩트 최대 = **P0 ①②** (추가 데이터 0, 사용자가 지적한 "누계 착시 + 진척률 착시" 동시 해소, 화면이 실제 MES "계획 대비 진도"처럼 작동).
- **구현 순서 제안:** ④제거 → ①Burn-up → ②페이스 진척률 → ③Top10 재정렬 → (P1) → (P2 데이터 수집 협의).
