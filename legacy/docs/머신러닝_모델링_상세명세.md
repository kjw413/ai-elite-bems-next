# 🧬 에너지 사용량 예측 모델 상세 명세 (준전문가용)

본 문서는 사내 BEMS(에너지 관리 시스템)에 적용된 에너지 예측 머신러닝 파이프라인의 데이터 수집, 전처리, 피처 엔지니어링, 모델링 알고리즘 및 평가 지표를 상세히 기술한 문서입니다. ADsP(데이터분석 준전문가) 수준의 배경지식을 가진 실무자가 전체 아키텍처와 방법론을 이해할 수 있도록 작성되었습니다.

---

## 1. 프로젝트 개요 및 목표

- **Task**: 다변량 시계열 회귀 분석 (Multivariate Time-Series Regression)
- **Target Variable (y)**: 공장별 일일 전력량(kWh), 연료량(N㎥), 용수량(ton)
- **목표**: 일 단위 생산량, 기상 조건, 캘린더 요인 등을 독립변수(X)로 하여 정상 상태의 에너지 사용량을 예측하고, 실제 사용량과의 오차율을 통해 **설비 및 공정의 이상 상태를 감지(Anomaly Detection)**하는 것.
- **버전 정보**:
  - **v5.3 (현재 운영 중)**: Quantile Regression (분위수 회귀, P05·P50·P95) + CQR 보정 기반 정상범주 밴드 예측, 재공품(WIP) 피처 반영, 웹 전체 재학습 통합. (v5.2와 동일 계열의 분위수 방식)
  - **v5.1 (레거시·롤백용 보존)**: Point Estimation (점 추정) 기반. 모델 레지스트리에 보존되어 즉시 롤백 가능.

---

## 2. 데이터 수집 및 파이프라인 구조

예측 모델은 4가지 도메인의 데이터를 병합하여 활용합니다.

1. **에너지 및 생산 실적 데이터 (MIS 연동)**
   - 원본 `RawDB_에너지.xlsx` 파일에서 데이터를 추출.
   - Wide-format (열: 날짜, 행: 항목) 데이터를 Long-format 시계열로 Transpose하여 전처리(`_maybe_transpose_energy` 함수).
2. **기상 데이터 (Open API 연동)**
   - 기상청(KMA) ASOS 일자료 API를 호출(`weather_sync_service.py`)하여 최근 누락분을 매일 실시간으로 동기화하고 `DB_weather.xlsx`에 로컬 캐싱.
3. **캘린더 및 휴일 데이터**
   - 주말 및 법정 공휴일 외에 '근로자의 날(5/1)' 등 실질적 공장 휴무일을 `DB_holiday.xlsx`로 관리 및 자동 보강.
4. **재공품(WIP) 데이터**
   - 각 공장·타겟별로 사전 수행된 상관분석(Correlation Analysis)을 통해 도출된 핵심 재공품(Shortlist) 항목만 선택적으로 병합.

---

## 3. 데이터 전처리 및 피처 엔지니어링 (Feature Engineering)

데이터 누수(Data Leakage)를 방지하고, 모델의 일반화 성능을 높이기 위해 아래와 같은 전처리 파이프라인이 적용됩니다.

### 3.1. 결측치 대치 (Imputation Strategy)

- **기상 데이터 결측 처리 (`fill_weather_gaps`)**: 도메인 지식(KMA 규약)을 반영하여 변수별로 다른 대치 기법 적용.
  - **일강수량**: 결측치는 무강수(0.0)로 Zero-fill.
  - **평균기온/상대습도**: 시계열 연속성을 가정하여 `ffill` (전방 채우기) 후 `bfill` 적용.
  - **일사량/일조시간**: Tree 기반 모델이 분기(Split) 시 NaN을 자체적으로 처리하도록 **결측치를 그대로 보존(NaN 보존)**. 두 변수가 상호 보완적으로 작용하도록 유도.

### 3.2. 종속변수 스케일링 (Target Transformation)

- 에너지 데이터 특성상 발생하는 우측 꼬리 분포(Right-skewed) 및 분산의 이분산성(Heteroscedasticity)을 완화하기 위해 **로그 변환(`log1p(y)`)** 을 수행한 공간에서 학습 진행.
- 추론 시에는 `expm1()`을 통해 원래 스케일로 역변환.

### 3.3. 파생 변수 생성 (Feature Extraction)

1. **시계열 지연 변수 (Lag & Rolling Features)**
   - `lag1`: 전일 사용량 (관성 반영)
   - `r7mean`: 과거 7일 이동 평균 (주간 베이스라인)
   - `intensity_lag1`: 전일 원단위 (`lag1 / 전일 생산량`). **Data Leakage 방지를 위해 당일 실측 원단위는 엄격히 배제.**
2. **기상 지수 (Meteorological Indices)**
   - **HDD (Heating Degree Days, 난방도일)**: `max(18 - 평균기온, 0)` - 겨울철 난방/보일러 부하 지표.
   - **CDD (Cooling Degree Days, 냉방도일)**: `max(평균기온 - 22, 0)` - 여름철 냉동기 부하 지표.
   - **THI (Temperature-Humidity Index, 불쾌지수)**: 기온과 상대습도를 결합한 체감 환경 지표.
3. **캘린더 플래그 (Calendar Events)**
   - 요일(`dow`), 월(`month`) 이외에 명절이나 장기 연휴에 따른 조업 변동성(Volatility)을 잡기 위해 플래그 변수 생성.
   - `is_pre_holiday`, `is_post_holiday`, `dist_to_h`(다음 휴일까지 거리), `consecutive_nonwork_before` 등.

### 3.4. 이상치 필터링 (Outlier Handling)

- `detect_special_events` 로직을 통해 믹스생산량이 30일 이동 중앙값의 10% 미만으로 급감하거나 1.0 미만인 '비정상 가동 중단일'을 학습 셋에서 제외. (Noise 제거 목적)

## 4. 모델링 및 앙상블 아키텍처

단일 모델의 과적합(Overfitting)을 방지하고 견고한 예측을 위해 **다중 앙상블 체계**를 운영합니다.

### 4.1. Base Estimators (3대 트리 부스팅 알고리즘)

정형 데이터 및 시계열 비선형성에 강점이 있는 GBDT 계열 알고리즘 채택.

- **LightGBM**: Leaf-wise 성장으로 빠른 속도와 높은 정확도 제공.
- **XGBoost**: Level-wise 성장 및 정규화(Regularization)를 통한 과적합 방지.
- **CatBoost**: Categorical 변수 및 작은 데이터셋에 대한 강건함.

### 4.2. Feature Subsets (관점별 앙상블 분기)

총 12개(3 Algorithms × 4 Subsets)의 Base Model을 병렬 학습합니다.

- **M1**: 생산량 + 시계열 Lag + 캘린더 요인 중심
- **M2**: 생산량 + 도일 지수(HDD, CDD) 중심
- **M3**: 생산량 + 기상 Raw Data(기온, 습도, 일사량) 중심
- **M4**: 생산량 + 체감 지수(THI, HDD, CDD) 중심

### 4.3. 가중치 최적화 (Ensemble Weight Optimization)

- 12개 예측 모델의 결과를 단순 산술 평균(Simple Average)하지 않고, **검증(Validation) 데이터셋**을 기반으로 최적의 가중치를 계산.
- `scipy.optimize.minimize`의 **SLSQP (Sequential Least SQuares Programming)** 알고리즘을 사용하여 가중치 합이 1이 되는 조건 하에 목적함수(v5.1: MAPE 최소화, v5.2: Pinball Loss 최소화)를 최적화합니다.

---

## 5. 학습 프로세스 및 OOT 검증 전략

미래 데이터에 대한 예측력을 담보하기 위해 **OOT (Out-Of-Time) Validation** 전략을 사용합니다.

- **Train Set (2021.01 ~ 2024.12)**: Base Model 학습. (WIP 데이터 반영 시점은 2023년부터)
- **Validation Set (2025.01 ~ 2025.12)**: 하이퍼파라미터 튜닝은 고정하되, 앙상블을 위한 **SLSQP 가중치(Weight) 학습**에 사용.
- **Test Set (2026.01 ~ 2026.04)**: 학습 및 검증에 전혀 관여하지 않은 최신 데이터를 활용하여 최종 성능 지표 산출.

---

## 6. 분위수 회귀 고도화 (v5.2/v5.3 — 적용 완료)

레거시 v5.1은 Point Estimation 기반이었으나, **현재 운영 중인 v5.2/v5.3** 모델은 **Quantile Regression**을 도입하여 예측의 신뢰 구간(정상범주 밴드)을 산출합니다.

- **목적 함수**: 각 알고리즘의 Loss Function을 Pinball Loss로 변경하여 학습 (예: LGBM `objective="quantile"`, XGB `objective="reg:quantileerror"`).
- **예측 분위수**: `Alpha = 0.05 (하한)`, `0.50 (중간값)`, `0.95 (상한)`
- **Quantile Crossing 보정**: 독립적으로 훈련된 하한/중간/상한 예측값의 단조성(Monotonicity)을 강제하기 위해, 앙상블 후 `np.sort(axis=0)`를 통해 물리적 모순(예: 하한이 상한보다 높은 경우)을 교정.
- **추가 평가 지표 (v5.2 전용)**:
  - **PICP (Prediction Interval Coverage Probability)**: 실제 값이 90% 신뢰 구간(P05~P95) 안에 들어온 비율. 모델의 이상감지 신뢰도 대변.
  - **MPIW (Mean Prediction Interval Width)**: 예측 구간의 평균 폭. 불확실성(Uncertainty)이 적을수록 폭이 좁음.
