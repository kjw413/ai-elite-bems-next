# 공장 전기·연료 사용량을 더 잘 예측하기 — 비전공자용 사례

> **한 줄 요약**: AI가 추천한 "에너지 사용량 예측에 도움될 것 같은" 80여 개 후보를, 그대로 믿지 않고 검증·실험·**현장 지식**을 합쳐서 13개만 추려냈습니다. 결과적으로 광주 공장 전기 예측 오차가 8.88% → 6.02%로 약 3%p 줄었고, 새로 가동된 라인 영향까지 따로 보정해서 추가 정확도 확보.

---

## 1. 우리가 하려던 일

빙그레 5개 공장(남양주1·남양주2·김해·광주·논산)에서 **내일 전기·연료·물을 얼마나 쓸지** 예측하는 AI 모델을 만들고 있습니다. 모델이 이미 보고 있는 정보:

- 오늘 만들 제품 총량
- 날씨 (기온, 습도, 일사량)
- 요일·월
- 어제 사용량

더 잘 예측하려면 "어떤 제품을 얼마나 만드는지" (재공품 = 제조 중인 품목별 양) 정보도 같이 넣어보기로 했어요. 문제는 **재공품 종류가 100개가 넘는다**는 것. 다 넣을 수는 없고, 진짜 도움 되는 것만 골라야 합니다.

---

## 2. 외부 AI한테 추천을 받았더니…

다른 웹 AI에 데이터를 주고 "예측에 좋을 만한 품목을 추천해줘" 했더니, 공장×에너지원마다 **6~8개씩, 총 80여 개**를 추천받았어요.

상관계수(r 값)도 0.5~0.84로 매우 높았습니다. **숫자만 보면 "와, 다 넣으면 되겠네!"** 싶죠. 그런데 그렇게 하면 안 됩니다.

---

## 3. 왜 상관계수만 보고 결정하면 안 되는가 — 4가지 함정

### 함정 1. "같이 다닌다고 친구는 아니다"

**비유**: 카페에 매일 두 사람이 동시에 와요. 친구일까요? …아닙니다. 둘 다 **옆 회사 직원**이고 점심시간이라 우연히 같이 옵니다.

**우리 문제에 적용하면**: 김해 욥닥터병이 전력과 r=0.78로 강해 보이지만, 사실은 둘 다 "오늘 공장이 가동되는가"라는 공통 원인의 결과일 뿐. 공통 원인 제거하고 다시 보면 r=0.19로 폭락. **보이는 신호의 75%가 가짜**였어요.

### 함정 2. "1000명한테 물으면 작은 의견도 다수가 된다"

데이터가 1,228일치면 r=0.07도 통계상 "유의미"하게 나옵니다. 변별력 없음.

### 함정 3. "이번 시험에 나온 문제로 다음 시험을 예측한다?"

외부 AI는 2021년부터 2026년 3월 데이터 **전체**를 보고 추천. 평가용 데이터까지 봤으니 "이 품목 넣으면 평가 잘 나오네!"는 당연. 시험 문제 미리 본 학생 같음.

### 함정 4. "재료가 너무 많으면 요리가 망한다"

80개 정보를 동시에 던지면 학생(모델)이 정작 중요한 5과목을 못 배웁니다.

---

## 4. 그래서 우리가 한 검증 — 다단계

### 1단계: "진짜 신호만 남기기"
- 공통 원인(생산량·요일·계절성) 제거 후 다시 상관 계산
- 학습 기간(2023~2024)만 사용
- **결과**: 80여개 → **23개**만 통과

### 2단계: "실제로 모델에 넣어보고 비교하기"
- A 모델: 재공품 없이 학습 → 평가 오차 측정
- B 모델: 재공품 추가 → 같은 평가 오차 측정
- 오차가 명확히 줄어든 경우만 채택

---

## 5. 결정적 전환점 — "현장 지식" 한 마디

여기까지 통계/실험만으로 정리했을 때, 사용자가 이런 말을 했어요:

> **"재공품 중에 공병(빈 병)은 넣는 게 좋지 않아? 생산량에 포함이 안 되는데 전력 소모가 많은 공정이야."**

이 한 문장으로 흐름이 바뀝니다. **공병 처리(세척·살균·이송)는 전기를 많이 쓰지만 "오늘 만든 제품 총량"에는 안 잡힙니다.** 통계가 못 본 hidden 정보를 도메인이 잡아준 것.

→ 사용자가 알려준 3개 공병 추가 후 광주 전력 **MAPE -2.48%p 큰 개선**.

**또 다른 현장 지식 정정**:
- "남양주1 시트에 적힌 재공품 중 일부는 사실 남양주2가 생산하는 거야" → 잘못된 라벨링 정정
- "농축유는 살균/농축 공정 부하 큼" → 남양주2·김해 연료/용수 spec에 반영
- "김해는 광주보다 전체 사용량 크고 공병 비중 작음" → 김해 적용 방식 조정

---

## 6. 또 하나의 문제 — "갑자기 새로 생긴 라인"

2026년 3월 25일부터 김해에 **신설 라인(223579)** 이 가동되기 시작했어요. 그런데:

- 학습 데이터(2023~2024)엔 223579 = 0 (없었음)
- 검증 데이터(2025)도 0
- 평가 데이터(2026)의 **3월 25일 이후만** 값이 있음

### AI 모델의 본질적 한계

모델이 학습할 때 어떤 정보가 **항상 0이면, 그 정보를 학습할 방법이 없어요.** 마치 시험 범위에 한 번도 안 나왔던 단원을 갑자기 시험에 내는 것과 같습니다. 학생이 아무리 똑똑해도 못 풉니다.

사용자도 직접 확인했어요: 223579를 학습 피처로 넣어봐도 결과가 똑같았습니다. 모델은 그 컬럼을 무시한 것.

### 해결책: **사후 보정**

학습으론 못 잡으니, **신설 후 운영 데이터로 영향 크기를 따로 추정**해서 모델 출력에 더하는 방식:

```
최종 예측 = 모델 예측 + (신설 라인 처리량 × 단위 부하 추정치)
```

신설 라인 가동 5주 분의 데이터로 단위 부하를 회귀 추정한 결과:
- 전력: 1단위 처리당 **+203 kWh**
- 연료: 1단위 처리당 **+38.5 N㎥**
- 용수: 거의 없음 (효과 0)

**보정 적용 후**: 김해 연료의 신설 라인 활성 구간 MAPE가 **15.49% → 9.85% (-5.63%p 큰 회복)**.

---

## 7. 최종 결과 — 채택된 13개 + 사후 보정

| 공장 | 적용된 학습 정보 |
|---|---|
| 광주 | 전기: 바나나병 / 연료·용수: 탈지분유·유크림·생크림 |
| 남양주2 | 전기/연료/용수: 주력 제품 4종 + (연료/용수만) 농축유 |
| 김해 | 전기: 공병 3종 / 연료·용수: 바나나맛우유믹스 + 농축유 + 수출용 |
| 남양주1 | 전기/연료/용수: 실제 생산품 2종 (시트 라벨링 정정 후) |
| 논산 | 용수만: 딸기맛우유믹스 + 바나나맛우유믹스 |

**+ 김해 신설 라인(223579) 운영 시 사후 보정 자동 적용**

## 8. 이 경험에서 배운 것 (전공 무관 교훈)

### "숫자는 답을 알려주지 않는다, 질문을 알려준다"
높은 상관계수는 "이거 봐!"라는 신호일 뿐, "이걸 써!"라는 결정이 아닙니다.

### "검증과 시험 데이터를 섞으면 점수가 가짜"
모델 만드는 데 쓴 데이터로 평가하면 안 됩니다.

### "통계는 보이는 것만 본다. 보이지 않는 것은 사람이 알려줘야"
공병이 별도 전력 부하라는 건 현장에서 일해본 사람만 압니다. AI가 아무리 똑똑해도 도메인 전문가의 한마디를 못 따라잡습니다.

### "실험 > 추정"
"이게 좋을 것 같다"보다 "실제로 넣고 빼고 비교해보자"가 정확합니다.

### "AI의 추천도 한 번 더 의심하라"
외부 AI 추천 80개 중 진짜 도움 되는 건 13개였습니다.

### "학습 못한 정보는 학습 못한 채로 둔다 — 운영에서 따로 해결"
신설 라인처럼 학습 데이터에 없던 신호는 모델 재학습으로 해결 안 됨. **운영 단계에서 별도 보정 기제가 필요**합니다.

---

## 9. 한 줄 정리

> **숫자(상관계수) + 실험(검증) + 사람(현장 지식) + 사후 보정(학습 못한 변화)** — 넷이 합쳐져야 진짜 좋은 모델이 됩니다.

---

## 부록: 전문 용어 vs 일상어

| 전문 용어 | 일상어로 풀면 |
|---|---|
| 상관계수(r) | 두 숫자가 같이 오르락내리락하는 정도 |
| Mix-adjusted correlation | 공통 원인 제거 후 본 진짜 관계 |
| p-value | "이게 우연일 확률" |
| Selection bias | 시험 문제 미리 보고 공부한 격 |
| Ablation | A vs B 빼고 넣고 비교 실험 |
| MAPE | 예측이 실제와 평균 몇 % 차이 |
| Concept drift | 모델 학습 후 세상이 바뀜 (신설 라인 등) |
| Post-hoc calibration | 사후 보정 — 모델 출력에 추가 항 더하기 |
| Feature | 모델에게 알려주는 정보 (재료) |
| Train/Val/Test | 공부용/모의고사/실전시험 데이터 |
