# [기능 명세서] FEMS (코드 기준)

이 문서는 현재 코드 기준으로 `app/main.py`와 실제 연결된 페이지/서비스를 설명하는 기능 명세서입니다.

---

## 1. 개요

- 시스템 명칭: 생산부문 에너지 관리시스템 (FEMS)
- 실행 방식: 로컬 Streamlit 웹 앱
- 데이터 저장소: MySQL
- 주요 목적: 에너지 실적 조회, 비교분석, 업로드 관리, AI 실적 보고서 지원

---

## 2. 전역 UI 구조

전역 화면 구성은 다음과 같습니다.

- 라이트 모드 단일 테마
- 사이드바 네비게이션
- 메인 탭 + 하위 탭 구조

사이드바 하단에는 아래 정보가 표시됩니다.

- 현재 시각
- 사용자명
- 전체 레코드 수
- 권한 배지 (🔓 root / 👁️ viewer)

권한은 비밀번호 입력 없이 클라이언트 IP 기반으로 자동 판정됩니다(호스트 PC → root, 외부 PC → viewer).

---

## 3. 메인 탭 구조

### 3.1 대시보드

현재 구현된 메인 대시보드의 핵심 기능은 다음과 같습니다.

- 7일간 생산량·원단위 추이 차트
- 월간 실적 전년비 현황 차트
- 유틸리티 원단위 현황 테이블
- 생산량 및 사용량 현황 테이블

핵심 특징:

- `전사`, `남양주`, `남양주1`, `남양주2`, `김해`, `광주`, `논산` 단위 조회
- 기준일에 따른 월간 진척률 계산
- 원단위/사용량/생산량 기준 전년 비교

### 3.2 생산실적

현재 상태:

- 화면 골격만 존재
- 일단위/월단위/연단위 탭 모두 플레이스홀더

### 3.3 에너지 모니터링

하위 메뉴 3개가 구현되어 있습니다.

1. 전력 사용량 (설비별 분해를 본 화면 내부 섹션으로 통합)
2. 연료·용수 사용량
3. 원단위

### 3.4 에너지 절감관리

현재 상태:

- `절감 계획 관리` 플레이스홀더
- `절감 실적 현황` 플레이스홀더

### 3.5 AI 에너지 분석

하위 탭은 아래와 같습니다.

1. `에너지 사용 예측`
   - viewer(외부 PC): `📈 예측 실행` / `📋 예측 이력` 두 탭만 노출됩니다. 예측은 실행되지만 DB에는 저장되지 않습니다.
   - 관리자(host PC): 위 두 탭에 더해 `⚙️ 모델 관리` 탭이 노출되며, v5 모델 메타데이터 / 재학습 트리거 / 기상청 API 수동 동기화 / 실측값 역채움이 가능합니다.
2. `에너지 실적 보고서`
   구현되어 있으며, DB 조회 기반 AI 보고서를 생성하고 저장합니다.

### 3.6 관리자 전용 탭

호스트 PC에서 접속하면(IP 기반 자동 판정) 아래 탭이 사이드바에 추가로 노출됩니다.

- `데이터 업로드`
- `변경 이력`

## 4. 대시보드 세부 기능

### 4.1 7일간 생산량·원단위 추이

- 기준일 기준 최근 7일 데이터 표시
- 생산량과 원단위를 이중축 차트로 렌더링
- 사업장 선택 지원

### 4.2 월간 실적 전년비 현황

- 데이터 유형 선택:
  - `원단위`
  - `사용량`
  - `생산량`
- 기준 연월 대비 전년 실적 비교

### 4.3 유틸리티 원단위 현황

- 전력, 연료, 용수 원단위 + 폐수/용수 비(폐수량/용수량, 소수점 2자리)를 비교 테이블로 표시
- 전년 실적, 전년 동월, 전년 동기 대비 비율 표시

### 4.4 생산량 및 사용량 현황

- 믹스 생산량과 주요 사용량 항목 비교
- 생산량은 높을수록 좋음, 원단위/사용량은 낮을수록 좋음 기준의 색상 표현 사용

---

## 5. 에너지 모니터링 기능

### 5.1 전력 사용량

- 일별 전력 추이
- 월별 전력 추이
- 전년 대비 비교
- KPI 카드:
  - 총 전력량
  - 냉동전력량
  - 공압기 전력
  - 전력 원단위
- 설비별 전력 구성(냉동설비/공압기/기타) 분해 섹션을 본 화면 내부에 통합

### 5.2 연료·용수 사용량

- 일별 연료 사용량
- 일별 용수/폐수 사용량
- 월별 비교
- 전년 대비 비교

### 5.3 원단위

- 전력/연료/용수 원단위 중 선택 (폐수 원단위 폐기)
- 일별/월별 원단위 추이
- 전년 대비 비교
- 폐수는 연료·용수 사용량 화면의 '공장별 폐수/용수' 섹션에서 관리

---

## 6. 데이터 관리 기능

### 6.1 데이터 업로드

- 지원 형식: `.xlsx`, `.xls`
- 관리자 전용 기능
- 업로드 시 아래 순서로 처리
  1. 파일 형식 검증
  2. 시트 및 컬럼 검증
  3. 데이터 정제
  4. DB 저장
  5. 감사 이력 기록
  6. 업로드 배치 이력 기록

업로드 결과 화면:

- 성공/실패 메시지
- 검증 오류 상세 표
- 업로드 이력 표

### 6.2 변경 이력

- 관리자 전용 기능
- 공장/기간 필터 제공
- 감사 로그 테이블 조회

---

## 7. AI 실적 보고서 기능

현재 구현된 AI 기능은 `에너지 실적 보고서`입니다.

기능 흐름:

1. 공장, 연도, 월 선택
2. 기존 저장 보고서 조회
3. `LangChain SQL Agent`가 DB를 직접 조회
4. LLM이 보고서 텍스트 생성
5. 결과를 `ai_reports` 테이블에 저장
6. 이후 같은 조건으로 재조회 가능

특징:

- 전사 또는 개별 공장 선택 가능
- 동일 공장/연월 보고서는 재생성 시 덮어쓰기

---

## 8. 권한 및 사용자 모드

- 기본 모드는 Viewer
- Admin 로그인 시 추가 메뉴 노출
- DB 접근도 관리자 계정과 조회 전용 계정으로 분리

## 9. 현재 미구현 범위

현재 메인 앱 기준으로 미구현 또는 플레이스홀더인 기능은 다음과 같습니다.

- 생산실적 상세 화면
- 절감 계획 관리
- 절감 실적 현황
- 웹 UI 내 에너지 사용 예측

---

## 10. 비고

- 현재 메인 내비게이션에는 데이터 내보내기 기능이 연결되어 있지 않습니다.
- 저장 및 집계의 기준 테이블은 `energy_daily`입니다.
- 공장 코드 `남양주`과 `전사`는 조회 시 계산되는 파생 그룹입니다.
