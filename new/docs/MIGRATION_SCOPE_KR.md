## 그대로 옮기기 어려운 부분

1. **Sites 주소에서 로컬 MySQL 직접 접속**  
   기술적으로 불가하며 보안상 허용해서도 안 됩니다. 실제 운영본은 서버 PC의 사내망 주소로 실행합니다.

2. **Streamlit 세션 상태의 1:1 복제**  
   React의 화면 상태로 대체했습니다. 사용자별 상태가 브라우저에 격리되므로 다중 사용자 안정성은 오히려 좋아집니다.

3. **기존 페이지의 모든 세부 옵션을 한 번에 완전 동일화**  
   핵심 운영 흐름과 주요 필터는 반영했습니다. 세부 옵션 중 생산실적의 연간 재활용 모드, 예측 이력 누락행 일괄 생성, 이벤트 수정·삭제 API, 목표설정 저장은 후속 API 엔드포인트 연결이 필요합니다. 화면 구조는 확장 가능하게 마련했습니다.

4. **실제 데이터 시각 검증**  
   개발 환경에는 사용자의 로컬 MySQL과 대용량 모델 파일이 없으므로, UI는 예시 데이터로 검증합니다. 서버 PC에서 첫 실행 후 단위·집계·날짜 경계를 기존 Streamlit 결과와 병행 비교해야 합니다.

## 운영 전 권장 검증

1. 전사·남양주 집계가 기존 화면과 일치하는지 확인
2. `production_daily.actual_qty` 오버레이에 따른 생산량·원단위 비교
3. 예측 단위(kWh/Nm³/ton)와 UI 표시 단위(MWh/천 Nm³/천 ton) 확인
4. 서버 PC와 다른 PC에서 admin/viewer 메뉴·API 권한 확인
5. 5~10명 동시 조회 시 DB connection·응답시간 측정
6. Excel 업로드를 복사 DB에서 먼저 검증한 후 운영 DB 적용
