# v5 에너지 예측모델 — 재공품 피처 선정 사례 연구

> **요약 한 줄**: 외부 AI가 추천한 80여 개 재공품 후보를 그대로 학습 피처로 쓰지 않고, 통계적 재검증·plant-target ablation·**도메인 지식 정정**·그리고 **신설 라인 사후 보정(drift correction)** 까지 합쳐 13개 spec으로 추렸습니다. 결과: 광주 전력 MAPE 8.88% → 6.02% (-2.86%p), 남양주2 용수 8.57% → 7.56% (-1.01%p), 신설 라인 영향 받는 김해 연료는 운영 시 사후 보정으로 추가 -5.6%p 회복.

---

## 1. 프로젝트 배경

- **시스템**: 빙그레 5개 공장(남양주1·남양주2·김해·광주·논산) 일별 전력/연료/용수 사용량 예측 (v5 모델, LightGBM+XGBoost+CatBoost 앙상블 × 4 모델 타입 = 12 모델 가중 평균)
- **기존 피처**: 생산량(`mix_prod_kg`) 로그변환, 타겟의 lag/rolling mean, 요일·월, HDD/CDD/THI 등 기상 파생지수
- **개선 시도**: 재공품(WIP) 품목별 일별 생산량을 학습 피처로 추가
- **출발점**: 외부 웹 AI가 작성한 두 산출물
  - `공장별_품목별_상관분석.xlsx` — 품목 vs 에너지원 raw Pearson r + p-value
  - `에너지예측_품목추천.xlsx` — plant-target별 6~8개씩 최종추천 (총 ~80개)

---

## 2. 문제 정의

> "외부 AI가 통계적으로 의미있다고 추천한 품목들을, 그대로 학습 피처로 넣어도 되는가?"

직관적 답: r 값 0.5~0.84로 매우 높고 p≈0이라 "와, 다 넣으면 되겠네!"
**실제 답: NO.** 4가지 함정 + 도메인 지식 검증 + 신설 라인 drift 까지 다뤄야 함.

---

## 3. 단순 상관계수의 4가지 함정

### 3-1. 통제변수 누락 — Raw r은 "공유 패턴"을 잡을 뿐

| 품목 | Raw r (전력) | Mix-adjusted r | 해석 |
|---|---|---|---|
| 김해 220006 (바나나맛우유믹스) | **0.773** | 0.19 | **거의 100% 생산량 효과** |
| 김해 220032 (욥닥터병) | **0.776** | 0.19 | 동일 |
| 광주 260014 (탈지분유) | 0.367 | **0.66** | 잔차에서 더 강한 진짜 신호 |

→ "이 품목이 생산되는 날 = 전체 생산량 많은 날 = 에너지 많이 쓰는 날". 모델은 이미 생산량을 학습 중이므로 **raw r이 높은 품목은 중복 정보**.

### 3-2. 큰 표본의 p-value 함정
표본 1,228일이면 r=0.07도 p<0.05. 통계적 유의성으로 거르면 거의 모든 품목 통과.

### 3-3. Selection bias
외부 AI는 2021-01 ~ 2026-03 전 기간에서 상관 계산 → train/val/test 모두 본 후 추천. 이걸 그대로 학습 피처로 박으면 **테스트 MAPE 낙관 편향**.

### 3-4. 카디널리티 폭발
5공장 × 3타겟 × 평균 6개 ≈ **80개 WIP 피처**. 일일 ~800행 데이터에서 약한 신호 다수는 노이즈로 작용.

## 4. 방법론 — 다단계 검증

### 4-1. 1단계: Mix-adjusted 잔차 상관 (Train 구간 한정)

[wip_residual_scorer.py](../app/predictive%20model/wip_residual_scorer.py)

```
1. Train 구간(2023-01-01 ~ 2024-12-31)만 사용 — selection bias 차단
2. 컨트롤 변수: log_mix_ton, log_mix_ton_lag1, dow, month
3. target_resid = target - OLS(target ~ controls)
4. item_resid   = item  - OLS(item  ~ controls)
5. Pearson r(item_resid, target_resid) → 진짜 잔차 신호
6. 임계: |r_resid| ≥ 0.20 AND nonzero_days ≥ 40 AND coverage ≥ 5%
```

**결과**: 99 후보 → **23개**만 통과.

### 4-2. 2단계: Plant-target 단위 Ablation

[wip_ablation_runner.py](../app/predictive%20model/wip_ablation_runner.py)

```
1. 동일 train 구간(2023-2024) 강제 — 데이터 길이 효과 통제
2. LightGBM 단일 모델, n_estimators=1500
3. baseline (WIP 없음) vs +WIP 두 조건 학습
4. val MAPE 차이(Δval)로 채택 결정 (임계: Δval ≤ -0.30%p)
```

### 4-3. 3단계: 도메인 지식 보정 (가장 큰 가치)

통계 컷오프만으론 잡지 못하는 hidden variable을 도메인 전문가가 알려줌:

| 도메인 정보 | 효과 |
|---|---|
| **공병(220032 욥닥터병, 220067/068 바나나병 상·하)은 mix_prod_kg에 미포함, 별도 전력 부하** | 통계만으론 r_resid=0.19로 탈락. **광주 전력에서 -2.48%p 큰 개선** |
| **남양주1 시트의 재공품 데이터는 사실 남양주2 생산품** | 잘못된 라벨링 정정. 남양주1엔 실 생산품(220006/184) 별도 채택 |
| **농축유(260015)는 살균/농축 공정에서 연료·용수 부하 큼** | 남양주2(주력 4종과 조합 시) 연료 -0.61%p, 용수 -1.01%p |
| **김해는 광주 대비 공병 부하 비중 낮음** | 김해 연료에 공병 단독으론 역효과. 농축유와 우유믹스 조합으로 발견 |

### 4-4. 4단계: 김해 단독 ablation (빠른 반복)

[modeling_v5.1_김해.py](../app/predictive%20model/modeling_v5.1_김해.py)

전공장 재학습 시간 부담을 줄이기 위해 김해만 단독 학습하는 스크립트. baseline + WIP 두 조건을 한 번에 비교. 김해 연료에서 **(바나나맛우유믹스 + 농축유 + 바나나맛우유믹스 수출용) 조합으로 test -2.96%p** 개선 발견.

### 4-5. 5단계: 신설 라인 사후 보정 (drift correction)

[analyze_223579_drift.py](../app/predictive%20model/analyze_223579_drift.py)

**문제**: 김해에 2026-03-25부터 신설 라인 223579 가동. 학습/검증 데이터엔 0 → 트리 모델이 학습 못함 → test 구간 MAPE 악화.

**해결 메커니즘**: 트리 모델의 본질적 한계 — 분산 0인 feature는 information gain 0 → 모델이 무시. 학습 다시 해도 신설 라인 신호는 부재.

**사후 보정**:
```
1. baseline 모델로 test 구간 예측
2. residual = actual - predicted
3. OLS (intercept 강제 0): residual = α × qty_223579
4. 운영 시: final_pred = model.predict(X) + α × qty_223579
```

**결과** (김해 단독 ablation 기준):
| Target | 활성 구간 baseline | 활성 구간 보정 후 | Δ | α (단위 부하) |
|---|---|---|---|---|
| 전력 | 7.56% | **5.84%** | **-1.72%p** | 203 kWh / 단위 |
| **연료** | **15.49%** | **9.85%** | **-5.63%p** | 38.5 N㎥ / 단위 |
| 용수 | 8.80% | 8.80% | ~0 | 효과 없음 |

운영 코드 [usage_prediction_v5_service.py](../app/services/usage_prediction_v5_service.py)에 `_apply_drift_correction()` 및 `DRIFT_CORRECTIONS` 상수로 반영.

## 6. 교훈 (Lessons Learned)

### 6-1. Raw correlation은 출발점일 뿐
모델이 이미 학습 중인 변수(생산량, 계절성)를 통제한 잔차 상관이 진짜 신호의 척도.

### 6-2. p-value의 함정
표본 크면 거의 모든 r이 유의함. 효과 크기와 실용적 의미가 진짜 척도.

### 6-3. Selection bias 경계
후보 선정과 평가는 다른 데이터로. Train 구간으로만 후보 점수, val/test로 채택 판정.

### 6-4. Ablation은 최후의 진실 판별기
잔차 통과해도 모델 학습에 도움 안 되는 경우 多 (김해 잔차 통과 3개 모두 reject). 잔차 미달이어도 도움 되는 경우 있음 (공병). **Plant-target 단위 baseline vs +WIP 직접 비교가 정답**.

### 6-5. 도메인 지식 > 통계
- 공병이 mix_prod_kg에 미포함 = 통계만 보면 영원히 못 찾았을 hidden load
- 시트 라벨링 오류(남양주1 데이터 = 사실 남양주2) — 데이터 검증으론 안 보임, 도메인만이 잡음
- 농축유 + 우유 조합 = 김해 연료의 정답
- 단독 통계 분석이 1주일 걸릴 만한 발견을 **현장 한 문장**이 1시간에 해결

### 6-6. 검증 임계의 한계
val MAPE Δ ≥ 0.30%p 기준은 합리적이나 절대적이지 않음. **김해 연료에서 val 미달이나 test에서 -2.96%p**처럼 test 결과와 도메인 일관성 종합 판단 필요.

### 6-7. 신설 라인은 학습 못함 — 사후 보정 필수
- 학습 데이터에 분산 0인 feature는 트리가 무시
- test 시 값이 있어도 예측 동일 (사용자가 직접 검증함)
- **새 라인 등장 = 모델 재학습으론 늦음. 운영 단계에서 사후 보정 기제 필요**

## 8. 프로세스 다이어그램

```
[외부 AI 추천 80개]
        ↓ Raw r + p-value
[1단계: Mix-adjusted 잔차 상관 (train 구간)]
        ↓ |r_resid|≥0.20, 99→23
[2단계: Plant-target Ablation (baseline vs +WIP)]
        ↓ Δval≤-0.30%p
[3단계: 도메인 지식 보정]
        ├── 공병 = mix 미포함 부하
        ├── 시트 라벨링 오류 정정
        └── 농축유 = 살균/농축 부하
        ↓
[4단계: 김해 단독 반복 (빠른 ablation)]
        ↓ 김해 연료 신규 조합 발견
[최종 13 specs] → modeling_v5.1.py → v5.1.pkl
        ↓
[5단계: 신설 라인 사후 보정 (운영 시점)]
        ↓ α × qty_223579
[운영 예측 = model.predict() + drift correction]
```

## 9. 한 줄 결론

> **상관계수는 hypothesis generator, 잔차+ablation은 hypothesis filter, 도메인 지식은 새로운 hypothesis source, 사후 보정은 학습 못한 신호의 마지막 방어선.** 넷 다 필요하고, 하나만으로는 안 된다.
