## 그대로 옮기기 어려운 부분

1. **Sites 주소에서 로컬 MySQL 직접 접속**  
   기술적으로 불가하며 보안상 허용해서도 안 됩니다. 실제 운영본은 서버 PC의 사내망 주소로 실행합니다.

2. **Streamlit 세션 상태의 1:1 복제**  
   React의 화면 상태로 대체했습니다. 사용자별 상태가 브라우저에 격리되므로 세션 간 상태 충돌 위험이 줄어들 것으로 예상하며, 동시 접속 검증은 아직 필요합니다.

3. **예측 입력 데이터 편집 그리드**  
   legacy의 생산량·기온 직접 수정(엑셀 복붙) 편집기는 이식하지 않기로 결정했습니다
   (2026-07-18 사용자 결정). 단일 예측 미리보기의 생산계획 kg 입력으로 대체합니다.
   임원 요약 One-Page 인쇄 뷰도 같은 날 이식 제외로 확정했습니다.

4. **실제 데이터 시각 검증**  
   개발 환경에는 실서버 MySQL과 대용량 모델 파일이 없으므로, 프런트 빌드와 DB 비의존
   백엔드 단위 테스트(47건), headless 렌더링 검증까지 완료했습니다. 서버 PC에서
   첫 실행 후 단위·집계·날짜 경계를 기존 Streamlit 결과와 병행 비교해야 합니다.

## 운영 전 권장 검증

서버 오픈·방화벽·사내망 접속·권한 점검의 실행 순서는 [RUN_GUIDE_KR.md](../RUN_GUIDE_KR.md)를 따른다.

1. 전사·남양주 집계가 기존 화면과 일치하는지 확인
2. `production_daily.actual_qty`와 광주 WIP 오버레이에 따른 생산량·원단위 비교
3. 예측 단위(kWh/Nm³/ton)와 UI 표시 단위(MWh/천 Nm³/천 ton) 확인
4. 서버 PC와 다른 PC에서 admin/viewer 메뉴·API 권한 확인 (메일 발송 카드 미노출 포함)
