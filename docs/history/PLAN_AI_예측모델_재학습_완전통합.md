# v5.3 웹 재학습 완전 통합

## 요약

- 관리자 버튼은 항상 최신 `energy_daily` 기반 v5.3 전체 재학습을 실행한다.
- 최근 30영업일은 테스트, 직전 60영업일은 검증/CQR, 이전 데이터는 학습에 사용한다.
- 새 모델은 모든 검증을 통과한 경우에만 자동 활성화하며, 실패 시 기존 모델을 유지한다.

## 핵심 구현

- v5.3 분위수 학습 코드를 공용 모듈로 분리하고 웹 워커와 `modeling_v5.3.py`가 함께 사용한다.
- 워커의 `skipped_quantile` 분기를 실제 학습으로 교체한다.
  - 5개 공장 × 전력·연료·용수 전체 학습
  - P05/P50/P95별 LGBM·XGB·CatBoost 및 M1~M4 구성 유지
  - Pinball 가중치, CQR `q_hat`, MAPE·PICP·MPIW 계산
  - 공장·대상·모델 유형·분위수 단위로 진행률 기록
- 후보 모델은 버전 경로에 임시 저장 후 다시 로드하여 검증한다.
  - 15개 공장·대상 스펙 모두 존재
  - 대상별 36개 모델과 가중치 구조 일치
  - 지표와 `q_hat`이 유한값이며 `P05 ≤ P50 ≤ P95`
  - 최신 테스트 행 샘플 예측 성공
- 검증 성공 후 모델 파일과 레지스트리를 원자적으로 교체한다. 실패한 후보는 삭제하고 기존 활성 경로와 학습일은 변경하지 않는다.
- 성공 후 타임스탬프 모델과 성능 보고서를 남기며, 활성 모델과 최근 이전 모델만 보존한다.
- 재학습된 모델에는 휴일 임시 사후 보정을 비활성화하고, 아직 학습 피처가 아닌 `223579` 드리프트 보정은 유지하는 메타데이터를 추가한다.

## 인터페이스와 상태

- `trigger_v5_retrain()` 시그니처는 유지하고 반환값과 상태 JSON에 `run_id`를 추가한다.
- 잠금도 `run_id + worker_pid` 소유권으로 관리해 이전 워커가 새 잠금을 점유하거나 남기는 경쟁 조건을 방지한다.
- 상태 모드는 `full_quantile`로 기록하고 `phase`, 분할 날짜, 현재 모델 유형·분위수, 후보/활성 경로를 노출한다.
- UI의 성공 표시는 실제 모델 활성화가 완료된 경우에만 표시한다.
- 설정 기본값:
  - `V5_QUANTILE_VALID_BDAYS=60`
  - `V5_QUANTILE_TEST_BDAYS=30`
  - `V5_FULL_RETRAIN_N_ESTIMATORS=3000`
  - 모델 보존 개수 2개
- 새 모델 스펙 버전을 추가하되 기존 v5.3 모델은 계속 로드할 수 있도록 하위 호환한다.

## 테스트 및 완료 기준

- 롤링 분할 경계, 데이터 부족, 공장·대상 누락 테스트
- 분위수 모델 구조·가중치·CQR·단조성 테스트
- 후보 검증 실패 시 기존 레지스트리와 모델 해시가 유지되는지 테스트
- 동시 요청, 죽은 워커, 오래된 잠금 및 `run_id` 소유권 테스트
- 작은 estimator와 합성 데이터로 워커 전체 흐름 통합 테스트
- 운영 환경에서 버튼 실행 후 `running → success` 진행, 새 버전 모델 활성화, 레지스트리 학습일 갱신 및 15개 대상 예측 성공 확인

## 가정

- 에너지 원천은 DB의 `energy_daily`, 재공품과 날씨는 현재 설정된 Excel 파일을 사용한다.
- 수동 요청은 학습 주기와 관계없이 항상 전체 재학습한다.
- 자동 활성화에는 구조·무결성·예측 검증을 적용하며 기존 모델 대비 성능 우위 조건은 적용하지 않는다.
