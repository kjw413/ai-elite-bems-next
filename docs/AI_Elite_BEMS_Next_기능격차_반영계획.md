## 2. 이번 반영 범위 (P0 — 사용자 지목 3건)

### P0-A. 메일 리포트 발송 UI/API ("메일 alert")
- `POST /api/v1/mail/send` (관리자 전용): `{period: daily|weekly|monthly, date?}`
  - `tools.mail` 빌더/발송 파이프라인 재사용 (legacy `_send_dashboard_report_mail` 동일 규칙)
  - 일간=기준일(미지정 시 빌더의 근무일 D-1 규칙), 주간=직전 완결 주, 월간=직전 완결 월
  - 설정 누락 시 누락 키 목록과 함께 503 반환, 발송 실패는 502
- 대시보드 상단(관리자에게만 노출): 주기 선택 + 발송 버튼 + 결과 표시

### P0-B. 에너지 사용량 — 기간 지정 조회 + 전년대비(연간 누계) 분석
- `GET /api/v1/energy`에 `date_from`/`date_to` 추가 (미지정 시 기존 30일 유지)
- 응답에 `yoy` 블록 추가: 기준연도 1~12월 금년/전년 사용량(4개 에너지원 전부)
- 화면: 조회 방식 토글(최근 30일/기간 지정) + "전년대비 사용량" 섹션
  - 금년 vs 전년 월별 막대 차트 (선택 에너지원)
  - 월별 테이블: 전년 실적·금년 실적·증감량·증감률 + **누계 행** (CSV 지원)

### P0-C. 대시보드 — 7일간 생산량·사용량 방향 비교
- `GET /api/v1/dashboard`의 `trend`에 연료·용수·폐수 일별 사용량 추가
- 화면: 전력/연료/용수/폐수 4분면 차트
  - 생산량 지수 vs 사용량 지수 (7일 중앙값=100, legacy `_median_index` 규칙)
  - 방향 신호: 생산↑사용↓(파랑)·생산↓사용↑(빨강) 마커, 중앙값 1% 이하 변동 무시
  - 상세 테이블 토글 + CSV (legacy 컬럼 구성 동일)
